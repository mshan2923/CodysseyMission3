# CodysseyMission3

| **비교 항목** | **Make** | **Zapier** |
| --- | --- | --- |
| **UI / UX** | 시각적 노드 기반의 워크플로우 편집기  | 리스트(단계별) 형태의 직관적인 인터페이스  |
| **설정 난이도** | 초기 학습 곡선은 있으나 세밀한 설계 가능  | 매우 직관적이고 설정이 간편함  |
| **연동 서비스 범위** | 광범위하며 고급 커넥터 기능 제공  | 매우 광범위하며 가장 많은 서비스와 연동 가능  |
| **무료 플랜 범위** | 약 1,000 Ops/월 (상대적으로 넉넉함)  | 약 100 Tasks/월 (상대적으로 제한적)  |
| **실행 로그 확인** | 단계별 데이터 상세 확인 가능  | Task History를 통한 실행 이력 확인  |
- **Make**
    - **장점**: 시각적인 노드 편집으로 복잡한 조건 분기(Router)를 설계하기 매우 유리함.
    - **단점**: 인터페이스가 복잡해 보여서 처음에 적응하는 시간이 필요함.
    - **적합 상황**: 조건 분기가 많은 복잡한 워크플로우를 구현하거나, 무료 플랜 내에서 더 많은 실행 횟수를 확보해야 할 때.
- **Zapier**
    - **장점**: UI가 단순해서 누구나 5분 안에 자동화를 시작할 수 있을 만큼 접근성이 높음.
    - **단점**: 복잡한 조건 분기가 들어가면 리스트 형태가 길어져 관리가 어려울 수 있음. 무료 플랜의 작업 제한(Task limit)이 타이트함.
    - **적합 상황**: 간단한 Trigger-Action 구조로 빠르게 자동화를 구축하고 싶을 때.

**반복 업무 정의**: 업무 효율을 위해 매일 확인해야 하는 중요 메일의 내용을 디스코드 채널로 실시간 알림을 받고, 구체적인 이력을 구글 시트에 아카이빙하는 작업.  
**선정 도구**: Zapier   
• 
**선정 이유**: 학습 초기 단계에서 가장 직관적인 UI를 제공하여 빠르게 워크플로우를 구현할 수 있고, 트리거와 액션 간의 연결 과정이 매우 명확하기 때문. 
**워크플로우 흐름**:

1. **Trigger (Gmail - New Email)**: 특정 라벨이 붙거나 중요 키워드가 포함된 새 메일 수신 시 작동.  
2. **Filter (조건 분기)**: 중요 메일로 분류된 건만 선별하여 불필요한 알림 차단.  
3. **Action 1 (Discord - Send Channel Message)**: 디스코드 포럼(채널)에 메일 제목과 요약 내용을 즉시 전송.  
4. **Action 2 (Google Sheets - Create Spreadsheet Row)**: 추후 이력 관리를 위해 메일 수신 시간, 발신자, 제목을 구글 시트의 새로운 행으로 자동 저장.
